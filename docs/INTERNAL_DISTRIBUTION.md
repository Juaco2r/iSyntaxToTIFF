# Internal distribution guide

For internal institutional users, the clean approach is:

1. Share `iSyntaxToTIFF-Windows-no-SDK.zip` from the GitHub Release or internal SharePoint.
2. Share the Philips SDK separately through an institution-approved internal SharePoint or software portal, only if permitted by the applicable SDK license.
3. Users configure the SDK path once from the application.

Do not commit the Philips SDK to GitHub.
Do not include the Philips SDK in public releases.
Do not share SDK download links or ZIP files with external users unless they are authorized under the applicable license.

External users should obtain the Philips Pathology SDK directly from Philips and configure it in the app.
